# PRD — 섹션 6~9: 가정사항, 범위 제외, 도메인 용어집, 미결 사항

> Index Power Forever | 작성일: 2026-03-21

## 6. 가정사항

작성 과정에서 명시적인 정보가 없어 설정한 가정 목록이다.

1. **무위험 수익률 고정**: 샤프 비율 계산 시 무위험 수익률을 연 4.5%(미국 10년 국채 기준)로 고정한다. 실시간 국채 수익률 API는 MVP 범위에서 제외한다.

2. **랭킹 ETF 목록 고정**: `/ranking` 화면의 ETF 목록은 사전 정의된 약 30~50개의 주요 ETF로 구성한다. 전체 미국 ETF 유니버스(수천 개)를 실시간으로 랭킹하는 것은 Yahoo Finance API 호출 비용 및 처리 시간 문제로 MVP에 포함하지 않는다.

3. **벤치마크 데이터 소스**: S&P 500은 ^GSPC 또는 SPY, NASDAQ 100은 ^NDX 또는 QQQ 심볼을 사용한다. 두 심볼이 동일하게 Yahoo Finance에서 조회 가능하다고 가정한다.

4. **타이밍 시뮬레이터 데이터**: 상위 N일 상승일 데이터는 백엔드에서 가격 히스토리로부터 계산한다. 별도의 이벤트 데이터베이스는 필요하지 않다.

5. **퇴직연금 시뮬레이터**: 적립식 투자의 수익률 계산은 월복리 방식으로 계산한다. 세금 효과(세액공제 등)는 MVP에서 제외하고 순수 운용 수익률만 비교한다.

6. **비용 시뮬레이터**: 복리 계산 시 비용은 매년 말 자산에서 차감하는 방식으로 단순화한다. 실제 ETF의 일별 비용 차감 방식과 다를 수 있으나 교육 목적으로 충분하다.

7. **인사이트 콘텐츠**: 책 『이길 수 있는 투자만 하라』의 구체적인 챕터 내용은 출간 전이므로, 공개된 정보와 일반적인 인덱스 투자 교육 내용을 바탕으로 작성한다. 출간 후 실제 내용에 맞게 업데이트가 필요할 수 있다.

8. **Yahoo Finance API 안정성**: Yahoo Finance의 비공식 API(cookie + crumb 방식)를 사용한다. API 구조 변경 시 YahooService를 업데이트해야 하는 위험이 있다.

9. **통화 단위**: 모든 금액은 USD로 표시한다. 환율 변환 기능은 MVP 범위에서 제외한다.

10. **ETF 설정일 이전 기간**: 요청한 비교 기간이 ETF 설정일 이전을 포함할 경우, ETF 설정일부터 비교를 시작하고 사용자에게 "이 ETF는 YYYY년 MM월에 설정되었습니다" 안내를 표시한다.

## 7. 범위 제외 (MVP 불포함)

다음 기능은 MVP에 포함하지 않으며, 향후 버전에서 검토할 수 있다.

| 제외 기능 | 제외 이유 |
|---|---|
| 로그인 / 회원가입 | 설계 원칙: 로그인 없이 즉시 사용 |
| 한국 ETF (KODEX, TIGER 등) | 미국 ETF 전용 서비스, Yahoo Finance API 제약 |
| 포트폴리오 리밸런싱 백테스팅 | SP500Simulator와 중복, 범위 초과 |
| 복수 ETF 동시 비교 | 단일 ETF vs 지수 비교에 집중 |
| 알림 기능 (가격 알림, 리포트 이메일) | 로그인 없음 원칙, 추가 인프라 필요 |
| 실시간 가격 (WebSocket) | 1시간 캐시 전략으로 충분, 비용 증가 |
| 사용자 정의 벤치마크 선택 | 항상 S&P 500 + NASDAQ 100으로 고정 |
| 소셜 공유 / 공유 링크 생성 | Phase 3 이후 검토 |
| 다국어 지원 (영어 등) | 한국어 전용 MVP |
| 배당 재투자 수익률 (Total Return) | 가격 수익률만 비교 (Yahoo Finance 데이터 기준) |
| 세금 효과 계산 | 순수 운용 수익률 비교에 집중 |
| 모바일 앱 (iOS / Android) | 웹 전용 MVP |
| 프리미엄 기능 / 구독 모델 | 전면 무료 서비스 |

## 8. 도메인 용어집

| 한국어 용어 | 영어 용어 | 정의 |
|---|---|---|
| 지수 | Index | S&P 500 또는 NASDAQ 100. 시장 전체의 성과를 대표하는 기준선 |
| 초과수익률 | Excess Return | ETF 누적수익률 - 지수 누적수익률. 양수면 지수를 이긴 것 |
| 누적 수익률 | Cumulative Return | 비교 시작 시점을 0%로 기준삼아 현재까지의 총 수익률 |
| 연환산 수익률 | CAGR | Compound Annual Growth Rate. 연평균 복리 성장률 |
| 최대 낙폭 | MDD | Maximum Drawdown. 이전 최고점 대비 최대 하락률 |
| 샤프 비율 | Sharpe Ratio | 위험 1단위당 초과수익. 높을수록 위험 대비 수익이 좋음 |
| 롤링 승률 | Rolling Win Rate | 임의의 시점에서 N년 보유 후 지수를 이긴 비율 |
| 롤링 수익률 | Rolling Return | 특정 보유 기간(N년)을 슬라이딩 윈도우로 계산한 수익률 |
| 벤치마크 | Benchmark | 비교 기준이 되는 지수. 이 서비스에서는 S&P 500, NASDAQ 100 |
| 총보수 | Expense Ratio | ETF 운용에 드는 연간 비용 비율(%). 낮을수록 유리 |
| 순자산 | AUM | Assets Under Management. ETF에 운용 중인 총 자산 규모 |
| 원리금보장형 | Principal-Guaranteed | 퇴직연금의 원금과 이자를 보장하는 보수적 상품 유형 |
| 액티브 ETF | Active ETF | 펀드매니저가 종목을 선택하는 ETF. 인덱스 ETF와 대비됨 |
| 인덱스 ETF | Index ETF | 특정 지수를 추종하는 패시브 ETF |
| 타이밍 실패 | Market Timing Failure | 상위 상승일을 놓쳐 장기 수익률이 크게 감소하는 현상 |
| 이김 / 짐 | Outperform / Underperform | ETF가 지수 대비 더 높은/낮은 수익률을 기록한 상태 |
| 프리셋 | Preset | 자주 비교되는 ETF 목록을 사전 정의해둔 것 |
| 설정일 | Inception Date | ETF가 처음 출시된 날짜 |

## 9. 미결 사항

현재 답변이 확정되지 않아 개발 시작 전 또는 개발 중 결정이 필요한 사항이다.

1. **Yahoo Finance API 안정성 대안**: 비공식 API인 Yahoo Finance가 차단되거나 구조가 변경될 경우, 대체 데이터 소스(Alpha Vantage, Polygon.io 등 유료 API)로 전환할 계획이 있는가?

2. **랭킹 화면 ETF 목록 확정**: 랭킹 화면에 포함할 ETF 목록(약 30~50개)을 구체적으로 확정해야 한다. 어떤 기준으로 선정할 것인가? (AUM 상위 N개? 직접 선정?)

3. **책 출간 전 인사이트 콘텐츠**: 책이 2026년 4월 출간 예정이고 서비스가 그 전에 오픈될 경우, `/insights`와 `/book` 화면의 콘텐츠를 어떻게 처리할 것인가?

4. **타이밍 시뮬레이터 데이터 범위**: 상위 N일 상승일 계산의 기준 자산은 S&P 500으로 고정하는가? NASDAQ 100도 포함할 것인가?

5. **퇴직연금 시뮬레이터 수익률 가정**: S&P 500 인덱스와 NASDAQ 100 인덱스의 예상 수익률을 어떻게 설정할 것인가? 과거 평균 수익률(S&P 500 약 10%, NASDAQ 100 약 13%)을 기본값으로 사용할 것인가?

6. **도메인 및 배포 URL**: 서비스의 공식 도메인이 확정되었는가? (예: index-power-forever.pages.dev 또는 커스텀 도메인)

7. **검색 로그 집계 주기**: 인기 검색 ETF TOP 10을 실시간으로 집계할 것인가, 아니면 주기적(예: 1시간)으로 집계할 것인가?

8. **MDD 계산 기준**: MDD 계산 시 일별 종가 기준으로 계산할 것인가, 장중 저가 기준으로 계산할 것인가? (Yahoo Finance 일별 데이터의 open/high/low/close 중 어느 것을 사용할 것인가?)
