## 제품명
Index Power Forever ("지수는 영원하다")

## 한 줄 요약
미국 ETF를 선택하면 S&P 500 / NASDAQ 100 지수와 성과를 비교 분석하여, 지수 투자의 장기 우월성을 시각적으로 증명하는 웹 서비스.

## 연계 도서
『이길 수 있는 투자만 하라』(이길수 저, 2026년 4월 출간)

## 핵심 메시지
- 대부분의 액티브/테마/섹터 ETF는 장기적으로 지수를 이기지 못한다
- "지수를 이길 수 있다"는 환상을 데이터로 깨뜨린다
- 확률적으로 유리한 구조(인덱스 투자)에 서라는 것이 진정한 "이길 수 있는 투자"

## 타겟 사용자
1. 투자 초보자: "이 ETF가 좋다던데?" → 지수와 비교해보고 판단
2. 장기 투자자: 지수 투자 확신을 데이터로 강화
3. 퇴직연금 가입자: 원리금보장형에 묶인 자산의 기회비용 확인
4. 투자 교육자/블로거: 콘텐츠 소재로 활용
5. 책 독자: 책 내용을 데이터로 직접 체험

## 핵심 기능 (7가지)

### 1. ETF 검색 및 종목 정보
- 미국 ETF 검색 (티커 또는 종목명)
- 기본 정보: 종목명, 티커, 운용사, 설정일, 총보수(Expense Ratio), 순자산(AUM), 추종 지수, 구성종목 TOP 10

### 2. 지수 대비 성과 비교 (핵심 기능)
- 수익률 비교 차트: ETF vs S&P 500 vs NASDAQ 100
- 초과수익률 차트: 양수(지수 이김)=녹색, 음수(지수 짐)=적색
- 성과 지표 테이블: 총수익률, CAGR, MDD, 샤프 비율, 변동성
- 기간 선택: 1Y / 3Y / 5Y / 10Y / 전체

### 3. 지수 이김/짐 분석 (차별화 기능)
- 지수를 이긴 기간 비율: "이 ETF가 S&P 500을 이긴 기간은 전체의 32%"
- 연도별 승패 기록: 매년 지수 대비 +/- 수익률 막대 차트
- 연속 이김/짐 기록
- 최근 추세 방향

### 4. 승률 대시보드
- 임의의 시점에 투자 → N년 보유 시 지수를 이길 확률
- 보유 기간별(1Y/3Y/5Y) 롤링 승률
- 36개월 롤링 수익률 차트

### 5. 비용의 복리 효과 시뮬레이터 (책 9장, 14장 연계)
- 동일 초기투자금, 동일 수익률에서 비용 차이가 만드는 장기 자산 격차
- 비교: 0.1%(인덱스) vs 1.0%(액티브) vs 2.0%(고비용 액티브)
- 사용자가 비용률/수익률/투자기간 직접 입력

### 6. 퇴직연금 인덱스 전략 시뮬레이터 (책 PART 4 연계)
- 비교: 원리금보장형(연 2~3%) vs KOSPI 200 인덱스 vs S&P 500 인덱스
- 실제 데이터: DC형 퇴직연금 평균 수익률 4.2%, 고수 그룹 38.8%
- 적립식 투자 결과 비교

### 7. 인기 비교 / 프리셋
- ARKK, XLK, VYM, GLD, TQQQ, EEM, XLE vs 지수
- 최근 많이 비교된 ETF TOP 10

## 기술 스택
- Frontend: Cloudflare Pages + Vue 3 CDN + Bootstrap 5 (ViewLogic, 빌드 없음)
- Backend: Cloudflare Workers + Hono
- Database: Cloudflare D1 (SQLite) — 캐시/집계 데이터
- Cache: Cloudflare KV
- 데이터 소스: Yahoo Finance API (cookie + crumb 인증)
- 벤치마크: ^GSPC (S&P 500), ^NDX (NASDAQ 100)
- 인증: 없음 (로그인 불필요)

## 핵심 계산 로직
```
누적 수익률 = (현재가 / 시작가 - 1) × 100
초과수익률 = ETF 누적수익률 - 지수 누적수익률
CAGR = (최종가/시작가)^(1/년수) - 1
MDD = (최저점 - 이전 최고점) / 이전 최고점 × 100
Sharpe = (CAGR - 무위험 수익률) / 연환산 변동성
롤링 N년 승률 = (N년 보유 후 지수 이긴 시작점 수) / 전체 시작점 수
```

## SP500Simulator와의 차이점

- 단일 ETF 선택 (복수 포트폴리오 아님)
- 항상 S&P 500 + NASDAQ 100 비교 (선택적 벤치마크 아님)

- D1 (SQLite) 사용 — 사용자 데이터 없이 캐시만
